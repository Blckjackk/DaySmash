Tambahkan animasi interaktif menggunakan library anime.js (https://animejs.com) ke website komunitas badminton ini untuk membuat pengalaman pengguna lebih dinamis dan tidak monoton. Import anime.js via CDN atau npm (animejs versi 3.x). ANIMASI YANG DIBUTUHKAN PER SECTION: 1. Hero Section - Saat halaman load, animasikan headline dengan efek "stagger" per kata atau per huruf — setiap kata muncul dengan translateY dari bawah ke posisi normal sambil fade in, dengan delay berurutan (staggered, delay 50-100ms antar elemen) - Gambar/ilustrasi shuttlecock di hero diberi animasi floating terus-menerus (translateY naik-turun pelan, loop infinite, easing easeInOutSine, durasi 2-3 detik) - Tombol CTA muncul dengan scale dari 0.8 ke 1 plus fade in, sedikit delay setelah headline 2. Counter/Statistik (jumlah member, turnamen, dll) - Animasikan angka dari 0 ke nilai akhir menggunakan anime.js dengan easing easeOutExpo, durasi sekitar 1.5-2 detik - Trigger animasi saat elemen masuk ke viewport (gunakan Intersection Observer untuk mendeteksi kapan section terlihat, baru jalankan animasi counter) 3. Card Jadwal Main dan Turnamen - Saat section masuk viewport, animasikan setiap card dengan stagger: translateY dari 30px ke 0 plus opacity dari 0 ke 1, delay antar card 100ms - Tambahkan animasi hover halus: scale ke 1.03 dan sedikit rotate (1-2 derajat) saat mouse hover, kembali ke normal saat mouse leave, dengan durasi cepat (200-300ms) 4. Progress Bar Slot Jadwal - Animasikan width progress bar dari 0% ke nilai aktual saat elemen masuk viewport, dengan easing easeInOutQuad, durasi 1 detik 5. Leaderboard Table - Setiap baris tabel muncul dengan animasi slide dari kiri (translateX dari -20px ke 0) plus fade in, stagger per baris dengan delay 80ms - Untuk top 3 (highlight emas/perak/perunggu), tambahkan sedikit efek "pulse" pada border atau background saat pertama muncul (scale 1 ke 1.05 lalu kembali, sekali saja) 6. Galeri/Masonry Grid - Setiap gambar di galeri animasikan dengan scale dari 0.9 ke 1 plus opacity dari 0 ke 1, stagger berdasarkan posisi grid (gunakan delay function anime.js: delay: anime.stagger(50, {grid: [kolom, baris], from: 'center'}) agar animasi menyebar dari tengah) 7. Navbar - Saat scroll ke bawah, animasikan navbar agar sedikit mengecil (height berkurang) dan background berubah jadi lebih solid/blur, dengan transisi halus menggunakan anime.js yang dipicu scroll event (throttle scroll listener agar performa tetap baik) 8. Section Transition (scroll-based) - Untuk setiap section baru yang masuk viewport, beri animasi reveal sederhana: translateY 40px ke 0 plus opacity 0 ke 1, durasi 600-800ms, easing easeOutCubic - Gunakan Intersection Observer untuk trigger animasi ini agar tidak membebani performa saat scroll KETENTUAN TEKNIS: - Gunakan custom React hook (misalnya useAnimeOnScroll) yang membungkus Intersection Observer + anime.js, agar bisa dipakai berulang di berbagai komponen tanpa duplikasi kode - Pastikan animasi hanya berjalan sekali saat elemen pertama kali masuk viewport (tidak berulang setiap scroll naik-turun), kecuali untuk animasi loop seperti floating shuttlecock - Tambahkan opsi untuk menghormati preferensi pengguna prefers-reduced-motion — jika aktif, nonaktifkan animasi non-esensial (seperti floating dan stagger) demi aksesibilitas - Jangan membuat animasi terlalu lambat atau berlebihan — durasi total animasi per elemen sebaiknya di bawah 1 detik agar tidak terasa lag, kecuali untuk floating/loop yang memang dimaksudkan pelan Pastikan semua animasi terasa halus (smooth) dan tidak mengganggu kecepatan loading halaman — gunakan lazy animation (animasi dijalankan hanya saat dibutuhkan/terlihat).
